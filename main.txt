Adopt ErrorOr result pattern, add AES cryptography, refresh tests & docs

Migrate the security helpers to an exception-free, ErrorOr-based API, add a
new AES encryption helper, and bring the tests and documentation in line.

Library
- CertificateFactory: every public method now returns ErrorOr<T> instead of
  throwing/returning raw values. The work is wrapped in the Protect helper
  from Forge.Next.Shared, which converts thrown exceptions into
  ErrorType.Unexpected errors (e.g. a wrong PFX password or a missing file is
  now a returned error rather than an exception). Both CreateSelfSigned
  overloads and GetForLocalhost gained a configurable keySize parameter
  (default 2048).
- CertificateAccess: every public method now returns ErrorOr<T>
  (ErrorOr<Success> for the mutating Add/Remove). FindByThumbprint returns an
  explicit Error.NotFound() when no certificate matches instead of null.
- AesCryptography (new): PBKDF2 key derivation (MakeAESKey) and AES-CBC
  encrypt/decrypt (AesEncrypt/AesDecrypt) producing a compact
  "iv:ciphertext" payload. Null arguments yield ErrorType.Validation errors;
  AesDecrypt maps internal failures back to the input via .Else so a wrong
  password never discloses the plaintext. Hash algorithm, output length and
  iteration count are configurable.

Project
- Bump AssemblyVersion 3.0.0.0 -> 3.1.0.0.
- Add the Forge.Next.Shared 3.0.0 package reference (supplies ErrorOr and the
  Protect extension).

Tests (tests/Forge.Next.Security.Tests)
- CertificateFactoryTests / CertificateStoreTests: rewritten against the
  ErrorOr API (asserting IsError / Value / FirstError.Type), plus new coverage
  for the configurable key size and the NotFound / Unexpected error paths.
- AesCryptographyTests (new): covers key derivation (determinism, salt /
  password / hash / iteration sensitivity, output length, null validation),
  encryption (iv:ciphertext format, empty input, random IV per call, null
  validation) and decryption (round-trip incl. custom parameters, empty input,
  malformed / undecodable input returned unchanged, and wrong-password not
  revealing the plaintext).

Documentation
- README.md: full rewrite documenting all three classes and every public
  method against the ErrorOr API, with an ErrorOr usage primer, per-method
  examples, the SAN build rules, an end-to-end example and platform notes.
